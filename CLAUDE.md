## What this repo is

A Flask web app that takes a work description from the user and returns a
Claude-generated motivational quote. It is the **target repo** for the
coding/review/test agents in `agent-prototype`.

## Running locally

```bash
pip install -r requirements.txt
cp .env.example .env       # add your ANTHROPIC_API_KEY
flask run                  # http://localhost:5000
```

Or with gunicorn (matches production):
```bash
gunicorn --bind 0.0.0.0:8000 app:app
```

Or via Docker:
```bash
docker build -t motivational-quote-app .
docker run -p 8000:8000 -e ANTHROPIC_API_KEY=... motivational-quote-app
```

## App structure

| Path | Purpose |
|---|---|
| `app.py` | Flask app ‚Äî routes: `GET/POST /`, `GET /health`, `GET /ping`, and `GET /version` |
| `templates/index.html` | Jinja2 template ‚Äî form + quote display |
| `Dockerfile` | Container image for Azure deployment |
| `Procfile` | Azure App Service startup command |

| `.agents/test_agent.py` | AI test generation + execution agent |
| `.github/workflows/ai-review.yml` | Runs review agent on every PR |
| `.github/workflows/test.yml` | Generates + runs tests on every PR |
| `.github/workflows/azure-deploy.yml` | Tests ‚Üí build ‚Üí deploy on push to main |

## Testing

Tests live in `tests/` and are **generated by the AI test agent** on each PR.
To run them locally:

```bash
pip install pytest pytest-cov
pytest tests/ --cov=app --cov-report=term-missing
```

Shared fixtures are in `tests/conftest.py`:
- `client` ‚Äî Flask test client with `TESTING=True`
- `mock_anthropic` ‚Äî patches `app.client` so no real API calls are made in tests

Coverage threshold: **70%** (enforced in CI).
