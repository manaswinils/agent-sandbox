# Implementation Plan: add a /health endpoint to app.py that returns JSON {"status": "ok"}

## Overview
Add a new `/health` HTTP GET endpoint to the Flask application in `app.py` that returns a JSON response `{"status": "ok"}` with a 200 status code. This is a lightweight health-check route useful for container orchestration (Azure Container Apps), load balancers, and monitoring systems to verify the application is running.

| *(none)* | No new files are needed |

## Files to Modify
| File | Change Required |
|------|----------------|
| `app.py` | Add a new route `/health` that returns `jsonify({"status": "ok"})` with HTTP 200 |
| `tests/conftest.py` | No change needed — the existing `client` fixture already supports testing new routes |
| `CLAUDE.md` | Update the "App structure" table to mention the `/health` endpoint alongside the existing `GET/POST /` route |

## Implementation Approach
1. **Import `jsonify`**: In `app.py`, add `jsonify` to the existing `from flask import ...` import statement, changing it to `from flask import Flask, jsonify, render_template, request`.
2. **Add the `/health` route**: Define a new function below the existing `index()` route:
   ```python
   @app.route("/health")
   def health():
       return jsonify(status="ok")
   ```
   This uses Flask's `jsonify` which automatically sets the `Content-Type` to `application/json` and returns HTTP 200 by default.
3. **Update `CLAUDE.md`**: In the "App structure" table, change the `app.py` description from `Flask app — single route GET/POST /` to `Flask app — routes: GET/POST / and GET /health` to reflect the new endpoint.

## Test Strategy
- **Unit test the `/health` endpoint**: Using the existing `client` fixture, write a test that sends `GET /health` and asserts:
  - Response status code is `200`
  - Response `Content-Type` is `application/json`
  - Response JSON body equals `{"status": "ok"}`
- **No mock needed**: The `/health` endpoint does not call the Anthropic API, so no `mock_anthropic` fixture is required.
- **End-to-end validation**: Run `flask run` locally and `curl http://localhost:5000/health` to confirm the JSON response. Also verify with `pytest tests/ --cov=app --cov-report=term-missing` that coverage remains above the 70% threshold.
- **Note on test generation**: Per `CLAUDE.md`, tests in `tests/` are auto-generated by the AI test agent on each PR. A manual test file can still be created for local validation, but the CI pipeline will generate its own tests.

## Risks and Assumptions
- **No authentication required**: The `/health` endpoint is intentionally public and unauthenticated, which is standard for health checks used by container orchestration platforms.
- **No dependency changes**: `jsonify` is built into Flask, so `requirements.txt` does not need modification.
- **Minimal blast radius**: The change adds a new route only; it does not modify any existing route or logic, so there is no risk to existing functionality.
- **Azure Container Apps**: If Azure is configured with a custom health probe path, it may need to be updated to point to `/health`. This is an infrastructure change outside the repo, but the endpoint will be ready to use.
